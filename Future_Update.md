## P0 — 核心功能缺失

> 影响日常使用体验的高频需求，应在 v2.0~v2.1 完成。

### 🔴 1. 标注历史与撤销栈

| 项目 | 内容 |
|------|------|
| **优先级** | P0 |
| **预估工时** | 2~3 天 |
| **涉及文件** | `core/annotation.py`, `ui/annotation_canvas.py`, `ui/main_window.py` |

**需求**：
- 每次标注操作（添加/删除/移动/缩放/改类）入栈
- `Ctrl+Z` 撤销，`Ctrl+Y` 重做
- 栈深度限制 100 步
- 切换图片时清空栈

**方案**：
```python
class UndoStack:
    def __init__(self, max_depth=100):
        self._stack = []
        self._ptr = -1
        self._max = max_depth
    
    def push(self, snapshot):       # 操作后保存当前图片标注快照
    def undo(self):                 # 恢复上一个快照
    def redo(self):                 # 恢复下一个快照
    def clear(self):               # 切图时清空
```

### 🔴 2. 实例分割完整支持

| 项目 | 内容 |
|------|------|
| **优先级** | P0 |
| **预估工时** | 5~7 天 |
| **依赖** | 标注历史栈完成后开工 |
| **涉及文件** | 全部模块 |

**改造范围**：

| 模块 | 改造内容 | 工时 |
|------|----------|------|
| 标注画布 | 多边形绘制/编辑/顶点拖拽 | 3 天 |
| 主窗口 | 任务类型切换、多边形快捷键 | 1 天 |
| 训练 | `task='segment'`，seg 预训练权重 | 0.5 天 |
| 推理 | mask 解码、轮廓提取、可视化 | 1.5 天 |
| 评估 | mask mAP 指标 | 0.5 天 |
| 数据集 | 格式校验（5字段 vs ≥7字段） | 0.5 天 |

**数据结构**：
```python
@dataclass
class YOLOShape:
    shape_type: str          # 'rect' | 'polygon'
    class_id: int
    points: list             # rect: [(x1,y1),(x2,y2)]  polygon: [(x,y),...]
```

**YOLO-Seg 标签格式**：`class_id x1 y1 x2 y2 ... xn yn`（坐标归一化 0~1，最少 3 点即 7 个数字）

**多边形交互**：
- 左键点击添加顶点
- 靠近首点（阈值 8px）且 ≥3 点时自动闭合
- 右键闭合当前多边形（≥3 点）或清空（<3 点）
- `P` 键切换多边形模式，`Esc` 取消
- 选中多边形后显示可拖拽顶点手柄

### 🔴 3. 大图分块加载

| 项目 | 内容 |
|------|------|
| **优先级** | P0 |
| **预估工时** | 1~2 天 |
| **涉及文件** | `ui/annotation_canvas.py`, `core/annotation.py` |

**需求**：>4000px 的图片标注时延迟明显，需要分块加载和渲染。

**方案**：只加载视口可见区域的图片瓦片 + 该区域标注框，缩放/平移时按需加载。背景用低分辨率缩略图填充不可见区域。

## P1 — 高优先级改进

> 显著提升效率的功能，应在 v2.2~v2.3 完成。

### 🟠 4. SAM 辅助标注

| 项目 | 内容 |
|------|------|
| **优先级** | P1 |
| **预估工时** | 3~4 天 |
| **涉及文件** | `core/sam_assist.py`（新）, `ui/annotation_canvas.py` |

**流程**：
1. 用户在图像上点击正样本点（框内）和负样本点（框外）
2. 调用 SAM 生成候选 mask
3. 提取 mask 轮廓 → 多边形
4. 用户微调后确认

**技术选型**：优先 MobileSAM（快速、轻量、ONNX 可部署），备选 FastSAM。

### 🟠 5. 视频标注支持

| 项目 | 内容 |
|------|------|
| **优先级** | P1 |
| **预估工时** | 3~4 天 |
| **涉及文件** | `core/video_annotation.py`（新）, `ui/main_window.py` |

**需求**：
- 视频帧序列标注（逐帧或跳帧）
- 关键帧标注 + 插帧自动标注（用 ByteTrack 跟踪传播）
- 已有模型预标注中间帧 → 人工修正

### 🟠 6. 标注快捷键自定义

| 项目 | 内容 |
|------|------|
| **优先级** | P1 |
| **预估工时** | 1 天 |
| **涉及文件** | `ui/main_window.py`, 配置文件 |

**需求**：用户可自定义标注快捷键，保存到 `~/.yolo_code/keybindings.json`。

## P2 — 中期功能扩展

> 扩展新任务类型和部署能力，v2.4~v3.0。

### 🟡 7. OBB 旋转框标注

- 四点旋转框标注（YOLOv8-OBB）
- 标签格式：`class_id x1 y1 x2 y2 x3 y3 x4 y4`
- 旋转框编辑（整体旋转、角点拖动）
- 预估工时：2~3 天

### 🟡 8. Pose 关键点标注

- 骨架定义配置（COCO 17 点等）
- 逐关键点标记 + 骨架连线预览
- 标签格式：`class_id x1 y1 conf1 x2 y2 conf2 ...`
- 预估工时：2~3 天

### 🟡 9. 标签格式互转

| 源 → 目标 | 用途 |
|-----------|------|
| YOLO 检测 → YOLO 分割 | 矩形自动转 4 点多边形 |
| YOLO 分割 ↔ COCO JSON | 跨工具协作 |
| LabelMe JSON → YOLO 分割 | 迁移旧标注 |
| YOLO 分割 → Mask PNG | 语义分割下游 |

- 预估工时：2 天

### 🟡 10. 导出功能扩展

- 分割数据导出：LabelMe JSON、Mask PNG
- 多边形简化开关（Douglas-Peucker，epsilon 可调）
- 最大点数限制
- 坐标归一化/反归一化选择
- 预估工时：1.5 天

## P3 — 长期优化

> 架构优化和锦上添花，时间充裕时处理。

### 🟢 11. 插件化架构

- 任务类型注册机制（detect / segment / obb / pose 插件化）
- 标注工具可插拔
- 导出格式可扩展
- 预估工时：5~7 天

### 🟢 12. 国际化支持

- 中英文界面切换
- 语言包机制（JSON 或 gettext）
- 预估工时：2 天

### 🟢 13. 数据增强预览

- Mosaic、MixUp、HSV 增强效果预览
- 训练前可视化确认
- 预估工时：1 天

### 🟢 14. 团队协作

- 标注任务分配
- 标注审核流程
- 多用户数据集合并
- 预估工时：5~7 天（适合作为 v3.0+ 大版本）
